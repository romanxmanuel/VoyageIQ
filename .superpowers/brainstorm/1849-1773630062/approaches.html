Three ways to approach this

All three cover everything you asked — they differ in order and risk level.

A

Content-first, search-second

Do first: Full-seed Manila, Sorsogon, Naga (+ Cebu, Davao). Overhaul all form copy and tier names. Add "Family trip" comfort/cost mode. Add budget input.

Do second: Wire hybrid global search once the content foundation is solid.

Pros

  • Philippines works great fast
  • Dad can plan immediately
  • Lower risk — no API keys needed yet

Cons

  • Still limited to seeded cities until Phase 2
B ★ Recommended

Everything in one sprint

Do together: Seed Manila + Sorsogon + Naga + Cebu + Davao fully. Rename all tiers and fix all copy. Add budget field + "easy travel" family toggle. Wire hybrid global search (Google Places autocomplete for typing, seeded data when matched, graceful fallback otherwise).

Pros

  • Ships as one coherent upgrade
  • No half-baked states for the user
  • Global search + Philippines both work day one

Cons

  • Needs a Google Places API key (free tier is generous)
  • Larger implementation sprint
C

No API keys — expand seed catalog broadly

Do: Seed 30+ destinations (Manila, Sorsogon, Naga, Cebu, Davao + Bangkok, Bali, Singapore, Lisbon, Rome, Cancun, etc.). Fix all copy. Add family mode + budget. No external API — everything stays self-contained.

Pros

  • Zero API keys, zero ongoing costs
  • Fast, reliable, works offline
  • Covers 90% of common trips

Cons

  • Still not "any city on earth"
  • More manual seeding work upfront

I recommend B — it ships the full vision in one clean pass. The Google Places API has a generous free tier and the autocomplete is what makes "any location on earth" feel real. Click your choice or reply in terminal.
